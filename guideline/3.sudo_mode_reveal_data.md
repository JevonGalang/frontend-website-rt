# 🔒 Bab 3: Verifikasi & Buka Kunci Data Sensitif (Sudo Mode)

Bab ini memuat panduan cara membuka data NIK dan No KK yang secara default di-masking (sensor huruf x) demi keamanan data warga. RT wajib melakukan verifikasi ulang menggunakan password login mereka untuk melihat data aslinya.

---

## 🔒 Verifikasi & Buka Kunci Data Sensitif (Sudo Mode)

Data `nik` dan `no_kk` yang dikembalikan dari API `/admin/resident` dan `/admin/datawarga` secara bawaan disensor (**masking**) dengan karakter `x`. Untuk melihat data lengkapnya, RT harus memasukkan password kembali.

### A. Reveal NIK Warga
Membuka sensor NIK milik individu warga tertentu.

* **Method & Route:** `POST /admin/reveal-warga/:id_warga`
* **Headers:** `Authorization: Bearer <token_jwt_rt>`
* **Request Body:**
  ```json
  {
    "password": "password_rt_yang_sedang_login"
  }
  ```
* **Response Sukses (200):**
  ```json
  {
    "nik": "3201234501010002"
  }
  ```

---

### B. Reveal No KK Keluarga
Membuka sensor nomor Kartu Keluarga (KK) milik satu keluarga tertentu.

* **Method & Route:** `POST /admin/reveal-resident/:id_family`
* **Headers:** `Authorization: Bearer <token_jwt_rt>`
* **Request Body:**
  ```json
  {
    "password": "password_rt_yang_sedang_login"
  }
  ```
* **Response Sukses (200):**
  ```json
  {
    "no_kk": "3201234567890002"
  }
  ```
