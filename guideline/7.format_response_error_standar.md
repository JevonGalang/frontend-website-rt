# ⚠️ Bab 7: Format Response Error Standar

Bab ini memuat daftar rujukan format payload JSON yang dikembalikan oleh server jika terjadi error pada input data (400), token habis/salah (403), atau penolakan otorisasi kepemilikan data (403).

---

## ⚠️ Format Response Error Standar

### A. Error Validasi Parameter / Input (400)
Terjadi jika data yang dikirim tidak lolos pengujian skema validasi masukan (misalnya, nomor KK kurang dari 5 karakter).

```json
{
  "success": false,
  "errors": [
    {
      "message": "No KK minimal 5 karakter",
      "path": ["noKK"]
    }
  ]
}
```

---

### B. Error Autentikasi / Token Tidak Valid (403)
Terjadi jika token JWT yang disematkan salah, tidak valid, atau sudah kedaluwarsa.

```json
{
  "message": "Token tidak valid"
}
```

---

### C. Error Akses Ditolak / Bukan Hak Milik (403)
Terjadi jika user mencoba mengakses atau memodifikasi data warga/keluarga lain yang bukan hak miliknya (Family-Gate authorization failed).

```json
{
  "pesan": "Akses ditolak, ini bukan data keluarga lu cuy!"
}
```
