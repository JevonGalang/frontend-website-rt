# 🪙 Bab 8: Sistem Keuangan, IPL (Iuran Bulanan), & Uang Kas RT

Bab ini memuat panduan integrasi sistem keuangan RT, yang membagi pencatatan keuangan menjadi **IPL (bulanan tetap)** dan **Uang Kas (insidental/sosial)**, pelacakan tunggakan warga (tracking), rapel iuran, buku kas besar (ledger), serta infografis dashboard landing page.

---

## 🧐 Perbedaan Sistem IPL vs Uang Kas

1. **IPL (Iuran Pengelolaan Lingkungan)**:
   * Sifatnya bulanan, nominal default **Rp200.000**.
   * Di dalamnya sudah mengalokasikan: Kebersihan/Sampah, Keamanan, Taman, dan Operasional RT.
2. **Uang Kas (Insidental/Sosial)**:
   * Sifatnya tidak berkala. Digunakan untuk kegiatan sosial atau insidental.
   * Kategori wajib salah satu dari: `"kematian"`, `"sosial"`, `"kegiatan"`, atau `"lainnya"`.
   * Membawa kolom `description` untuk input manual nama kegiatannya (misal: "Bantuan Warga Sakit", "Iuran 17 Agustusan").

## 👥 Endpoint Khusus Warga (`/resident`)

### A. Bayar Iuran IPL (Support Rapel)
Warga mengupload bukti transfer pembayaran IPL bulanan. Untuk rapel (misal 1 tahun sekaligus di bulan Januari), frontend mengirimkan array bulan yang ingin dibayar.

* **Method & Route:** `POST /resident/pay-ipl`
* **Headers:** 
  * `Authorization: Bearer <token_jwt_warga>`
  * `Content-Type: multipart/form-data`
* **Request Body (Multipart Form-Data):**
  * `months`: Stringified JSON array bulan (1-12), contoh: `"[1, 2, 3]"` (untuk bayar bulan Jan, Feb, Mar).
  * `year`: Tahun target, contoh: `2026` (tipe: `Integer`).
  * `amount`: Total uang yang ditransfer, contoh: `600000` (tipe: `Integer`).
  * `file`: Berkas bukti transfer (PDF/JPG/PNG, max 5MB).
* **Response Sukses (200 OK):**
  ```json
  {
    "response": 200,
    "output": {
      "message": "Pembayaran IPL pending berhasil dicatat masbro",
      "payment_ids": [10, 11, 12]
    },
    "message": "Bukti pembayaran IPL berhasil diunggah masbro, menunggu approval bendahara"
  }
  ```

---

### B. Bayar Uang Kas (Insidental)
Warga menyetorkan sumbangan sosial atau iuran kegiatan insidental.

* **Method & Route:** `POST /resident/pay-kas`
* **Headers:** 
  * `Authorization: Bearer <token_jwt_warga>`
  * `Content-Type: multipart/form-data`
* **Request Body (Multipart Form-Data):**
  * `amount`: Nominal sumbangan, contoh: `50000` (tipe: `Integer`).
  * `category`: Kategori kas. Harus berupa salah satu dari: `"kematian"`, `"sosial"`, `"kegiatan"`, `"lainnya"`.
  * `description`: Nama kegiatan/keterangan iuran, contoh: "Iuran Santunan Kematian Pak Slamet" (tipe: `String`).
  * `file`: Berkas bukti transfer (PDF/JPG/PNG, max 5MB).
* **Response Sukses (200 OK):**
  ```json
  {
    "response": 200,
    "output": {
      "fieldCount": 0,
      "affectedRows": 1,
      "insertId": 3
    },
    "message": "Bukti pembayaran Kas berhasil diunggah masbro, menunggu approval bendahara"
  }
  ```

### B. Approve / Reject Pembayaran IPL
Bendahara menyetujui atau menolak bukti bayar IPL warga. Jika status disetel `"diterima"`, sistem otomatis mencatat nominal tersebut sebagai pemasukan (`in`) di buku kas RT (Ledger).

* **Method & Route:** `PATCH /admin/finance/approve-ipl/:payment_id`
* **Headers:** `Authorization: Bearer <token_jwt_bendahara_atau_rt>`
* **Request Body:**
  ```json
  {
    "status": "diterima" 
  }
  ```
  * `status`: Wajib `"diterima"` atau `"ditolak"`.
* **Response Sukses (200 OK):**
  ```json
  {
    "response": 200,
    "output": {
      "fieldCount": 0,
      "affectedRows": 1,
      "changedRows": 1
    },
    "message": "Pembayaran IPL berhasil di-set diterima masbro!"
  }
  ```

---

Bendahara menyetujui atau menolak bukti bayar Kas/Sumbangan sosial warga. Jika status disetel `"diterima"`, nominal masuk otomatis ke Buku Kas Ledger sebagai pemasukan.

* **Method & Route:** `PATCH /admin/finance/approve-kas/:payment_id`
* **Headers:** `Authorization: Bearer <token_jwt_bendahara_atau_rt>`
* **Request Body:**
  ```json
  {
    "status": "diterima"
  ```

  ```json
      "fieldCount": 0,
  ```

### D. Mencatat Pengeluaran Kas RT (Ledger)
Bendahara mencatat pengeluaran uang kas RT (misal beli sapu keamanan, bayar gaji satpam, santunan kematian).

* **Method & Route:** `POST /admin/finance/expense`
* **Headers:** `Authorization: Bearer <token_jwt_bendahara_atau_rt>`
* **Request Body:**
  ```json
  {
    "amount": 1500000,
    "sourceType": "keamanan",
    "description": "Gaji bulanan Security RT 02"
  }
  ```
  * `sourceType`: Harus berupa salah satu dari: `"kebersihan"`, `"keamanan"`, `"taman"`, `"operasional_rt"`, `"kematian"`, `"sosial"`, `"kegiatan"`, `"lainnya"`.
* **Response Sukses (200 OK):**
  ```json
  {
    "response": 200,
    "output": {
      "fieldCount": 0,
      "affectedRows": 1,
      "insertId": 24
    },
    "message": "Pengeluaran kas RT berhasil dicatat cuy!"
  }
  ```

Bendahara mencatat transaksi dana masuk non-iuran bulanan (seperti donasi warga, sumbangan sukarela, subsidi, sponsorship, dll.).

* **Method & Route:** `POST /admin/finance/income`
* **Headers:** `Authorization: Bearer <token_jwt_bendahara_atau_rt>`
* **Request Body:**
  ```json
  {
    "amount": 500000,
    "sourceType": "donasi",
    "description": "Donasi fogging warga Blok B"
  }
  ```
  * `sourceType`: Harus berupa salah satu dari: `"donasi"`, `"donasi_sukarela"`, `"subsidi"`, `"sponsorship"`, `"hibah"`, `"lainnya"`.
* **Response Sukses (200 OK):**
  ```json
  {

    "output": {

      "affectedRows": 1,
      "insertId": 25
    },
  ```
