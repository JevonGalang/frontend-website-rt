# 🛠️ Bab 2: Alur Pengelolaan Data & Skenario Pendaftaran Warga 

Bab ini memuat panduan lengkap tentang alur input data kependudukan secara berurutan, skenario pendaftaran yang benar (biar database kita gak dipenuhi rumah fiktif!), serta kueri-kueri terkait untuk verifikasi pendaftaran mandiri warga, pengelolaan pengaduan, pengajuan, dan pengumuman.

---

## 🧐 Logika Dasar Database Kita

Biar gak salah paham, ini struktur hubungan data kita di database:
* **1 Rumah (`house`)** ➔ Ditempati oleh **1 Keluarga (`family/KK`)**
* **1 Keluarga (`family/KK`)** ➔ Berisi **Banyak Warga (`warga`)**

> 📢 **PENTING UNTUK TIM FRONTEND!**
> Satu rumah itu bisa diisi satu keluarga. Satu keluarga isinya bisa banyak orang. Jangan tiap daftarin satu orang, lu bikinin rumah baru sama KK baru! Emang warga kita jomblo tajir melintir semua apa? 😭

---

## 🛠️ Skenario Pendaftaran yang Benar (UI/UX Guidelines)

Frontend **HARUS** membagi pendaftaran warga menjadi **3 Skenario** ini di UI/UX aplikasi:

### 🌟 SKENARIO A: RT Mendaftarkan Anggota Keluarga Baru (KK-nya sudah terdaftar)
> **Contoh:** RT mau daftarin anak baru lahir, atau istri yang baru pindah gabung KK suami yang sudah terdaftar di sistem.

* **Alur API oleh Frontend:**
  1. **JANGAN** panggil `POST /admin/house` ❌
  2. **JANGAN** panggil `POST /admin/resident` ❌
  3. **LANGSUNG TEMBAK** ➔ **`POST /admin/datawarga`** (Sebagai RT) ✅
     * Kirim parameter `fammilyId` dan `houseId` dari KK yang sudah terdaftar (RT tinggal pilih nama kepala keluarga dari dropdown di UI).
     * Status warga ini otomatis langsung **diterima** (aktif).

---

### 🌟 SKENARIO B: RT Mendaftarkan Keluarga Baru (KK Baru & Rumah Baru) (DIREKOMENDASIKAN - ONE STEP)
> **Contoh:** Ada satu keluarga baru yang baru pindah ke lingkungan RT kita.

* **Alur API oleh Frontend:**
  1. **LANGSUNG TEMBAK** ➔ **`POST /admin/register-family`** (Sebagai RT) ✅
     * Kirim data rumah, nomor KK, dan data diri Kepala Keluarga sekaligus dalam satu request body.
     * Response otomatis mengembalikan `houseId`, `familyId`, `kepalaKeluargaId`, serta **username & password login sementara** untuk warga tersebut.
  2. Berikan username dan password sementara tersebut ke Kepala Keluarga.
  3. Kepala Keluarga login, lalu menambahkan istri/anak secara mandiri lewat **SKENARIO C** (sehingga backend otomatis menyambungkannya ke KK dan Rumah yang sama tanpa membuat data baru).

> 📌 **Catatan:** Skenario sekuensial manual (Step 1 s/d 4 di bawah) masih bisa digunakan sebagai alternatif lama, namun sangat disarankan menggunakan Skenario B One-Step di atas agar database aman dari rumah fiktif/duplikasi KK.

---

### 🌟 SKENARIO C: Warga Mendaftarkan Anggota Keluarganya Sendiri (Pendaftaran Mandiri)
> **Contoh:** Akun warga login ke website, lalu mendaftarkan anggota keluarga barunya secara mandiri.

* **Alur API oleh Frontend:**
  1. **LANGSUNG TEMBAK** ➔ **`POST /resident/datawarga`** (Sebagai Warga) ✅
  2. **JANGAN** kirim `fammilyId` dan `houseId` di request body! ❌ Server otomatis mengambil data KK warga yang sedang login demi keamanan.
  3. Data warga yang dikirim berstatus **diterima** (langsung aktif). Warga baru ini akan **langsung muncul** di list anggota keluarga (`GET /resident/getmyfamily/:id`) tanpa menunggu verifikasi/persetujuan dari RT.

## 📋 Detail Kueri API Pengelolaan RT

### METODE UTAMA (REKOMENDASI) — One-Step Family Registration (`POST /admin/register-family`)
Mendaftarkan Rumah baru, Kartu Keluarga baru, Kepala Keluarga, sekaligus membuatkan akun login warga dalam satu kali panggil secara aman (menggunakan database transaction).

* **Method & Route:** `POST /admin/register-family`
* **Headers:** `Authorization: Bearer <token_jwt_rt_atau_sekretaris>`
* **Request Body:**
  ```json
  {
    "blok": "A",
    "nomor": 12,
    "alamat": "Jl. Mawar No. 12",
    "statusRumah": "pribadi",
    "noKK": "3201234567890001",
    "nik": "3201234567890002",
    "nama": "Budi Santoso",
    "jenisKelamin": "Laki-laki",
    "tglLahir": "1980-05-12",
    "statusHidup": "Hidup",
    "noHp": "081234567890",
    "umur": 46
  }
  ```
* **Response Sukses (200 OK):**
  ```json
  {
    "response": 200,
    "output": {
      "houseId": 12,
      "familyId": 8,
      "kepalaKeluargaId": 25,
      "account": {
        "username": "keluarga_8",
        "temporaryPassword": "xT8$2aQ9"
      }
    },
    "message": "Pendaftaran kepala keluarga dan pembuatan akun berhasil dilakukan secara otomatis!"
  }
  ```
  > 📌 **Tindakan Frontend:** Tampilkan username dan password sementara di atas kepada RT agar bisa diserahkan langsung ke Kepala Keluarga yang baru terdaftar.

---

### STEP 1 — Tambah Data Rumah (`POST /admin/house`)
* **Method & Route:** `POST /admin/house`
* **Request Body:**
  ```json
  {
    "blok": "B",
    "nomor": 15,
    "alamat": "Jl. Kamboja No. 15",
    "status": "pribadi"
  }
  ```
  * `status`: Hanya boleh `"pribadi"` atau `"kontrak"`.
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "pesan": { "insertId": 3 },
      "token": null
    },
    "message": "data nya sudah terkirim"
  }
  ```
  > 📌 **Tindakan Frontend:** Simpan `insertId` (id rumah) untuk dipakai pada Step 2 dan 3.

---

### STEP 2 — Tambah Data KK (`POST /admin/resident`)
* **Method & Route:** `POST /admin/resident`
* **Request Body:**
  ```json
  {
    "noKK": "3201234567890002",
    "home": 3,
    "KepalaKeluarga": 1
  }
  ```
  * `home`: ID Rumah dari Step 1.
  * `KepalaKeluarga`: ID Warga Kepala Keluarga (dapat diisi `1` untuk inisialisasi sementara, lalu di-update nanti).
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "pesan": { "insertId": 5 },
      "token": null
    },
    "message": "masuk dengan sempurna"
  }
  ```
  > 📌 **Tindakan Frontend:** Simpan `insertId` (id keluarga/family) untuk dipakai pada Step 3.

---

### STEP 3 — Tambah Anggota Keluarga Warga (`POST /admin/datawarga`)
* **Method & Route:** `POST /admin/datawarga`
* **Request Body:**
  ```json
  {
    "nik": "3201234501010002",
    "nama": "Ahmad Subarjo",
    "jenisKelamin": "Laki-laki",
    "tglLahir": "1994-05-12",
    "statusHidup": "Hidup",
    "noHp": "081298765432",
    "umur": 30,
    "fammilyId": 5, // ⚠️ AWAS TYPO! Pake double 'm' (fammilyId)
    "houseId": 3
  }
  ```
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "pesan": { "insertId": 10 }
    },
    "message": "masuk dengan sempurnaaa"
  }
  ```

* **Response Sukses (200):**

---

### STEP 6 — Update Status Pengaduan (`PATCH /admin/pengaduan/:id`)
> RT/Sekretaris menyetujui atau menolak laporan pengaduan warga.

* **Method & Route:** `PATCH /admin/pengaduan/1`
* **Headers:** `Authorization: Bearer <token_jwt_rt_atau_sekretaris>`
* **Request Body:**
  ```json
  {
    "status": "disetujui"
  }
  ```
  * ⚠️ **Aturan Status:** Nilai `status` hanya boleh diisi `"pending"`, `"disetujui"`, atau `"ditolak"`.
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "fieldCount": 0,
      "affectedRows": 1,
      "info": "",
      "serverStatus": 2,
      "warningStatus": 0,
      "changedRows": 1
    },
    "message": "status pengaduan berhasil diupdate"
  }
  ```

### STEP 7.2 — Mengelola Pengumuman RT (`/admin/announcement`)
> RT/Sekretaris dapat membuat, mengubah, menghapus, atau melihat pengumuman untuk warga.

#### A. Membuat Pengumuman (`POST /admin/announcement`)
* **Headers:** `Authorization: Bearer <token_jwt_rt_atau_sekretaris>`
* **Request Body:**
  ```json
  {
    "judul": "Gotong Royong",
    "isi": "Ayo bersihkan lingkungan hari Minggu besok jam 08.00 WIB."
  }
  ```
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "pesan": {
        "affectedRows": 1,
        "insertId": 1
      }
    },
    "message": "pengumuman berhasil dibuat masbro"
  }
  ```

#### B. Mengubah Pengumuman (`PATCH /admin/announcement/:id`)
* **Headers:** `Authorization: Bearer <token_jwt_rt_atau_sekretaris>`
* **Request Body (Bisa kirim judul/isi saja):**
  ```json
  {
    "isi": "Ralat jam: Mulai jam 09.00 WIB."
  }
  ```
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "pesan": {
        "affectedRows": 1
      }
    },
    "message": "pengumuman berhasil diperbarui masbro"
  }
  ```

#### C. Menghapus Pengumuman (`DELETE /admin/announcement/:id`)
* **Headers:** `Authorization: Bearer <token_jwt_rt_atau_sekretaris>`
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "pesan": {
        "affectedRows": 1
      }
    },
    "message": "pengumuman berhasil dihapus masbro"
  }
  ```

#### D. Melihat Pengumuman (Sebagai Pengurus) (`GET /admin/announcement`)
* **Headers:** `Authorization: Bearer <token_jwt_rt_atau_sekretaris>`
* **Response Sukses (200):**
  ```json
  [
    {
      "id": 1,
      "judul": "Gotong Royong",
      "isi": "Ralat jam: Mulai jam 09.00 WIB."
    }
  ]
  ```

### 🔍 5. Fitur Pencarian Warga Panel RT

Membantu RT & Sekretaris mencari data warga dengan cepat dalam situasi darurat (seperti kematian atau keributan) untuk mengetahui identitas lengkap, asal keluarga (KK), dan lokasi rumah mereka.

#### A. Melakukan Pencarian Warga (`GET /admin/resident/search?q=<query_pencarian>`)
* **Headers:** `Authorization: Bearer <token_jwt_rt_atau_sekretaris>`
* **Query Params:** `q` (bisa diisi nama, NIK asli, No KK asli, blok rumah, nomor rumah, alamat, atau status hidup).
* **Response Sukses (200):**
  ```json
  [
    {
      "warga_id": 45,
      "nik": "320123*********2",
      "nama": "Budi Santoso",
      "jenis_kelamin": "Laki-laki",
      "tgl_lahir": "1980-05-12",
      "status_hidup": "Hidup",
      "no_hp": "081234567890",
      "umur": 46,
      "family_id": 8,
      "family_nokk": "320123*********1",
      "house_id": 12,
      "house_blok": "A",
      "house_nomor": "12",
      "house_alamat": "Jl. Mawar No. 12",
      "house_status": "pribadi",
      "status": "diterima"
    }
  ]
  ```
  * ⚠️ **Keamanan Data:** Informasi sensitif seperti NIK asli dan No KK asli tetap di-masking secara default di hasil pencarian. RT harus memanggil endpoint `reveal` untuk melihat isi lengkapnya.
