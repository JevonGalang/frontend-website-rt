# 👥 Bab 4: Fitur & Endpoint Warga (Role: `warga`)

Bab ini memuat panduan lengkap tentang fitur yang ditujukan untuk warga, mulai dari penggantian password wajib pertama kali, pengaduan keluhan, pengajuan surat pengantar RT, pengamatan pengumuman, dan pendaftaran anggota keluarga mandiri.

---

## 👥 Fitur & Endpoint Warga (Role: `warga`)

### A. Ganti Password Pertama Kali (Wajib)
Warga wajib mengganti password saat pertama kali login menggunakan password sementara yang digenerate oleh Ketua RT.

* **Method & Route:** `PATCH /resident/password`
* **Headers:** `Authorization: Bearer <token_jwt_sementara>`
* **Request Body:**
  ```json
  {
    "newPassword": "PasswordBaruWarga123!"
  }
  ```
* **Response Sukses (200):**
  ```json
  {
    "pesan": "Password berhasil diperbarui masbro!"
  }
  ```

---

### B. Ambil Data Anggota Keluarga
Mengambil seluruh daftar anggota keluarga di dalam KK yang bersangkutan.

* **Method & Route:** `GET /resident/getmyfamily/:id_family`
* **Headers:** `Authorization: Bearer <token_jwt_warga>`
* **Response Sukses (200):**
  ```json
  [
    {
      "warga_id": 10,
      "nik": "320xxxxxxxxxx002",
      "nama": "Ahmad Subarjo",
      "jenis_kelamin": "Laki-laki",
      "tgl_lahir": "1994-05-12",
      "status_hidup": "Hidup",
      "no_hp": "081298765432",
      "umur": 30,
      "family_id": 5,
      "house_id": 3,
      "house_blok": "B",
      "house_nomor": "15",
      "house_alamat": "Jl. Kamboja No. 15",
      "house_status": "pribadi"
    }
  ]
  ```
  > 🛡️ **Proteksi Keamanan:** Warga biasa hanya diizinkan memanggil endpoint ini jika `:id_family` sesuai dengan `family_id` milik akunnya sendiri. Jika berbeda, backend mengembalikan status **`403 Forbidden`** (Akses ditolak).

### C. Buat Pengaduan Warga (Tabel `report`)
Membuat laporan keluhan/pengaduan fasilitas lingkungan.

* **Method & Route:** `POST /resident/pengaduan`
* **Headers:** `Authorization: Bearer <token_jwt_warga>`
* **Request Body:**
  ```json
  {
    "isi": "Selokan mampet depan pos ronda",
    "jenis_pengaduan": "Fasilitas Publik"
  }
  ```
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "fieldCount": 0,
      "affectedRows": 1,
      "insertId": 1
    },
    "message": "pengaduan berhasil terkirim"
  }
  ```
  > 📌 **Catatan:** Kolom `status` otomatis di-set `'pending'` secara bawaan di database.

### H. Pendaftaran Anggota Keluarga Mandiri (Tabel `warga` - Langsung Aktif)
Warga mendaftarkan anggota keluarga baru. Data langsung terdaftar resmi dan berstatus aktif (diterima) tanpa menunggu verifikasi/persetujuan dari RT.

* **Method & Route:** `POST /resident/datawarga`
* **Headers:** `Authorization: Bearer <token_jwt_warga>`
* **Request Body:**
  ```json
  {
    "nik": "3201234567890012",
    "nama": "Siti Aminah",
    "jenisKelamin": "Perempuan",
    "tglLahir": "1998-07-20",
    "statusHidup": "Hidup",
    "noHp": "08122334455",
    "umur": 28
  }
  ```
  > 🔒 **Keamanan:** `family_id` dan `house_id` tidak perlu dikirim di request body. Backend otomatis mengambil dari KK warga yang login untuk mencegah kecurangan.
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "pesan": {
        "fieldCount": 0,
        "affectedRows": 1,
        "insertId": 15
      },
      "token": null
    },
    "message": "pendaftaran anggota keluarga berhasil, data langsung aktif masbro!"
  }
  ```

### J. Melakukan Vote Karyawan Terbaik (`POST /resident/vote`)
Mengirimkan pilihan vote warga untuk karyawan terbaik. Setiap akun hanya diizinkan memilih **maksimal 1 kali**. Percobaan vote kedua kali otomatis ditolak.

* **Method & Route:** `POST /resident/vote`
* **Headers:** `Authorization: Bearer <token_jwt_warga>`
* **Request Body:**
  ```json
  {
    "karyawanId": 1
  }
  ```
* **Response Sukses (200):**
  ```json
  {
    "response": 200,
    "output": {
      "pesan": {
        "fieldCount": 0,
        "affectedRows": 1,
        "insertId": 1,
        "info": "",
        "serverStatus": 2,
        "warningStatus": 0,
        "changedRows": 0
      },
      "token": null
    },
    "message": "Vote lu berhasil dimasukkan masbro, mantap!"
  }
  ```
* **Response Error (Double Vote / Sudah Pernah Vote) (400):**
  ```json
  {
    "pesan": "error: Lu udah ngevote masbro, cuma bisa 1 vote per akun!"
  }
  ```

---
